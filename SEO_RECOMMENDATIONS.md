# Рекомендации по SEO

Последнее обновление: 2026-06-29.

Production: https://mycamdesk.com

## Текущее состояние live-сайта

- Публичные страницы открыты для индексации.
- `robots.txt` доступен и разрешает публичные разделы.
- Закрыты приватные зоны: `/admin`, `/cabinet`, `/auth`, `/api`.
- `sitemap.xml` доступен и содержит главную, каталоги, гайды, статьи, профили, вакансии, услуги, товары, резюме и связки модель-оператор.
- На основных публичных страницах есть `title`, `description`, `canonical`.
- Есть `llms.txt` для LLM-краулеров.
- Есть OG-image и favicon.
- Есть JSON-LD: `WebSite`, `Organization`, `CollectionPage`, `Article`, `BreadcrumbList`, `FAQPage`, `Product`, `ProfilePage`, `JobPosting`, `Service`.
- SEO URL сделаны транслитом с коротким ID:
  - статьи: `/articles/{shortId}-{translit-title}`;
  - вакансии: `/rabota/{title}-vebcam-studii-{city}-{shortId}`;
  - услуги: `/uslugi/{title}-dlya-vebcam-{city}-{shortId}`;
  - товары: `/tovar/{title}-{category}-dlya-vebcam-modeli-{city}-{shortId}`;
  - резюме: `/resume/{role-title}-vebcam-model-{city}-{shortId}`;
  - модель-оператор: `/svyazki/{model/operator}-ischet-{model/operator}-webcam-{title}-{city}-{shortId}`.
- Фильтрованные каталоги в основном канонизируются на базовый раздел, например `/vacancies?city=...` -> canonical `/vacancies`.
- Тематическая лента статей `/articles?topic=...` имеет отдельный canonical и может индексироваться как тематическая страница.

## Что проверено 2026-06-29

- Главная: `200`, indexable, canonical `https://mycamdesk.com`, JSON-LD есть.
- `/articles`, `/vacancies`, `/services`, `/products`, `/resumes`, `/model-operator`, `/authors`, `/stories`, `/money`, `/safety`, `/work`, `/links`: `200`, indexable, canonical есть.
- Детальные страницы статей, вакансий, услуг, товаров, резюме: `200`, indexable, canonical есть, JSON-LD есть.
- `/auth/signin`, `/auth/signup`, `/cabinet`, `/admin`, `/search`: закрыты через `noindex` или редирект на вход.
- Быстрый внешний поиск `site:mycamdesk.com` не показал стабильной индексации. Это может означать, что сайт еще не попал в индекс или попал частично.

## Главные проблемы

1. Поисковая индексация пока не подтверждена.
   Нужно подключить Google Search Console и Яндекс.Вебмастер, добавить sitemap и вручную отправить главную, каталоги и первые гайды на индексацию.

2. Нет полноценной страницы `/guides`.
   Сейчас гайды существуют как детальные SEO-страницы, но пункт "Гайды" в меню ведет на один конкретный гайд. Нужна витрина всех гайдов.

3. `manifest.json` содержит старый бренд.
   Сейчас в manifest есть `WebcamExpert Journal` / `WE Journal`, а проект уже `MyCamDesk`. Нужно заменить название, short_name и цвета.

4. Noindex-страницы отдают canonical на главную.
   Для `/auth`, `/cabinet`, `/admin`, `/search` лучше не отдавать canonical на `/`, чтобы не создавать лишний сигнал для поисковиков.

5. `JobPosting` для вакансий нужно усилить.
   Уже есть базовая разметка, но желательно добавить `baseSalary`, нормализованный `employmentType`, более чистое описание и аккуратную географию для удаленных вакансий.

6. `Product` schema для товаров нужно проверить на абсолютные image URL.
   Для Google/Yandex лучше отдавать абсолютные ссылки на картинки, цену, валюту, состояние, наличие, продавца и срок актуальности.

7. UGC-страницы могут быть SEO-слабыми, если контент сырой.
   Нужно предусмотреть SEO title/description для статей или автоматически улучшать их из заголовка, рубрики и лида.

## Приоритетный план

1. Подключить Google Search Console и Яндекс.Вебмастер.
2. Отправить `https://mycamdesk.com/sitemap.xml`.
3. Запросить индексацию главной, каталогов и первых гайдов.
4. Исправить `manifest.json` на MyCamDesk.
5. Сделать страницу `/guides` как каталог всех гайдов.
6. Добавить `/guides` в меню и sitemap.
7. Убрать canonical на главную у noindex-страниц.
8. Усилить `JobPosting` для вакансий.
9. Усилить `Product` schema для товаров.
10. Добавить SEO-поля или автоулучшение SEO title/description для статей.

## Как должна работать страница `/guides`

- Это не лента и не UGC-статьи, а редакционная база знаний.
- На странице должны быть категории: `Новичкам`, `Деньги`, `Безопасность`, `Работа`, `Студии`, `Оборудование`, `Продвижение`, `Модель и оператор`.
- Карточка гайда должна содержать:
  - рубрику;
  - название;
  - короткое описание;
  - кому полезно;
  - кнопку `Читать`.
- Каждая карточка ведет на отдельный SEO-лендинг `/guides/...`.
- Блок "Гайды" в меню должен вести на `/guides`, а не на один конкретный гайд.

## Важные правила

- Вакансия должна быть доступна без логина, не закрыта robots.txt и без `noindex`.
- Если вакансия удалена или ушла в архив, ее нужно убрать из sitemap или отдать корректное архивное состояние.
- В `JobPosting.title` не стоит добавлять зарплату, город, капс, рекламные символы и лишние призывы.
- Для удаленной вакансии нужно явно указывать удаленный формат в schema.
- Google для вакансий рекомендует Indexing API, чтобы быстрее сообщать о новых и измененных вакансиях.
- Товарные страницы лучше размечать максимально полно: цена, валюта, наличие, состояние, изображение, продавец, срок актуальности цены.
- Schema должна соответствовать видимому контенту на странице.
- SEO-гайды и UGC-статьи нельзя смешивать: статьи дают живой опыт, гайды закрывают поисковый спрос.
